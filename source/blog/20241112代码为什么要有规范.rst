代码为什么要有规范?
==========================

再简单的代码经过不断的迭代和堆砌，最终也会变得臃肿和复杂。

规范的意义就在于在代码不断复杂的过程中，依然保证代码的可读性和可修改性。


对代码进行合理的分层
------------------------

::
    
    # 错误示例
    func1(func2(func3(func4())))
    dict = {1:{2:{3:{4:5}}}}
    # 正确示例
    func1(
        func2(
            func3(
                func4()
            )
        )
    )

    dict = {
        1:{
            2:{
                3:{
                    4:5
                }
            }
        }
    }

对代码进行分层的目的有:

1. 快速定位嵌套函数的起止结构
2. 代码需要修改时，方便按行修改，新增和删除
